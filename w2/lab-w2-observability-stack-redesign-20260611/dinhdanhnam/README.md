# Observability + AIOps Stack Redesign Submission

Read `architecture-target.mmd` first for the target-state data flow and ownership boundaries, then read `cost-model.md` to see how the design cuts spend from `$42,000/month` to `$16,551/month` while keeping the run-rate below the `$25,200/month` requirement. The submission is intentionally hybrid: OpenTelemetry, VictoriaMetrics, Loki, Tempo, Grafana, and Alertmanager handle core observability in the company environment, while Grafana Cloud IRM and Statuspage remain SaaS for paging and customer communication. The main evidence trail is embedded where reviewers need it: `components.md` maps each component to pain points and public docs, `cost-model.md` includes unit formulas and pricing URLs, the ADRs cite official Loki/Tempo/OpenTelemetry/Grafana/SRE sources, `migration-plan.md` shows dual-run and 30-minute rollback gates, `risks.md` names the six concrete risks, and `FINDINGS.md` includes the A7 Loki POC plan.
